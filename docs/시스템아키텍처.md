## 2. 컴포넌트 아키텍처

### 2.1 Frontend 구조

```
frontend/src/
├── App.jsx                  # 라우팅 + AppContext Provider
├── main.jsx
├── pages/                   # 11개 페이지 컴포넌트
│   ├── Home.jsx             # 서비스 소개 + CTA
│   ├── Consent.jsx          # 개인정보 동의
│   ├── AreaSelect.jsx       # 진단 영역 선택
│   ├── Onboarding.jsx       # JB금융 연동 (페르소나 선택)
│   ├── Survey.jsx           # 설문 진단 (동적 렌더링)
│   ├── PreDiagnosis.jsx     # 사전 질문 (동적 단계)
│   ├── Dashboard.jsx        # 재무 현황 대시보드
│   ├── Diagnosis.jsx        # 진단 결과 (레이더/바 차트)
│   ├── Simulation.jsx       # 현금흐름 예측 (분포형 차트)
│   ├── Recommend.jsx        # 맞춤 추천 (액션카드+상품)
│   └── Chat.jsx             # AI 상담 (메시지 UI)
├── api/
│   └── apiClient.js         # Axios 인스턴스 + API 함수
├── data/
│   ├── surveyData.js        # 설문 문항 25개 + 점수 기준
│   └── mockPersonas.js      # 시연용 페르소나 3종
└── styles/
    └── global.css
```

**AppContext 데이터 흐름**

```
Onboarding → profile ────────────────────────────────→ Simulation
                                                      → Recommend
                                                      → Chat
AreaSelect → selectedAreas ──→ Survey ──→ surveyScores → Diagnosis
                                                       → Recommend
PreDiagnosis → profile 보완 (나이/직업/은퇴나이/건강/이벤트)
```

### 2.3 AI Agent 아키텍처

```
사용자 요청
    │
    ▼
┌─────────────────────────────────────────┐
│          Diagnosis Agent                │
│  · 규칙 기반 점수 계산                   │
│  · 재무/이벤트/소비/건강 4개 점수        │
│  · 동연령 비교 + 위험 영역 식별          │
│  (Claude API 미사용)                    │
└─────────────────────────────────────────┘
    │ DiagnosisResult
    ▼
┌─────────────────────────────────────────┐
│         Simulation Agent                │
│  [1단계] 룰 기반 현금흐름 계산 (35년)    │
│    · 물가/임금/의료비 상승률 적용        │
│    · 생애이벤트 반영                    │
│    · 신뢰구간(±1σ) 계산                │
│                                         │
│  [2단계] Claude AI 인사이트 생성        │
│    · 핵심 리스크 1가지                  │
│    · 시급한 행동 1가지                  │
│    · 3~4문장 평문 출력                  │
└─────────────────────────────────────────┘
    │ SimulationResult
    ▼
┌─────────────────────────────────────────┐
│          Recommend Agent                │
│  [1단계] 규칙 기반 액션카드 생성         │
│    · 취약 영역별 최대 5개               │
│                                         │
│  [2단계] RAG 벡터 검색                  │
│    · jhgan/ko-sroberta-multitask 임베딩 │
│    · ChromaDB 유사도 검색               │
│    · 실제 상품 4개 + 가상 상품 3개      │
│                                         │
│  [3단계] Claude 최종 선별               │
│    · 후보 7개 → 최적 3개 선택           │
│    · 추천 이유 생성 (각 2줄)            │
└─────────────────────────────────────────┘
    │ RecommendResult
    ▼
┌─────────────────────────────────────────┐
│            Chat Agent                   │
│  · free 모드: 자유 질문 응답            │
│  · proactive 모드: 5단계 가이드 상담    │
│  · 진단 결과 컨텍스트 주입              │
│  · SSE 스트리밍 지원                   │
└─────────────────────────────────────────┘
```


### 3.2 RAG 데이터 흐름

```
상품 문서 (*.md)
    │
    ▼ 최초 1회 빌드 (rebuild_vectorstore)
jhgan/ko-sroberta-multitask
    │
    ▼ 임베딩 생성
ChromaDB (chroma_db/)
    │
    ▼ 쿼리 시 검색 (search_products)
사용자 상황 → 자연어 쿼리
    │
    ▼
유사도 검색 (top-k)
    │
    ▼
Claude API → 최종 3개 선별
```

### 3.3 MLOps 데이터 흐름

```
KReIS 10차 원본 (data/raw/*.csv)
    │
    ▼ [featurize] build_features.py
naro_features.csv (8,736명 × 39컬럼)
    │ 타겟: finance_score, health_score, leisure_score, relation_score
    ▼ [train] train.py
XGBoost vs RandomForest 5-Fold CV
    │
    ▼
models/{target}_best.pkl (4개)
reports/model_performance.csv
    │
    ▼ [explain] shap_explain.py
SHAP TreeExplainer
    │
    ▼
shap_{target}_{summary|bar}.png (8개)
shap_importance.csv
```

## 4. 기술 스택

| 레이어 | 기술 | 버전/용도 |
|--------|------|----------|
| **Frontend** | React | 19 |
| | Vite | 번들러 + 개발서버 |
| | react-router-dom | v7, SPA 라우팅 |
| | Recharts | 데이터 시각화 (레이더/바/라인 차트) |
| | Axios | API 통신 (timeout 60s) |
| | lucide-react | 전문 아이콘 시스템 |
| **Backend** | FastAPI | Python 비동기 웹 프레임워크 |
| | Uvicorn | ASGI 서버 |
| | Pydantic | 데이터 검증 및 스키마 |
| **AI** | LangChain | Agent/LLM 체인 |
| | langchain-anthropic | Claude API 연동 |
| | Claude claude-sonnet-4-6 | 생성 AI 모델 |
| **RAG** | ChromaDB | 로컬 벡터 데이터베이스 |
| | sentence-transformers | jhgan/ko-sroberta-multitask |
| **ML** | XGBoost | 그래디언트 부스팅 |
| | scikit-learn | RandomForest, 교차검증 |
| | pandas / numpy | 데이터 처리 |
| | joblib | 모델 직렬화 |
| **MLOps** | DVC | 데이터·모델 버전 관리 |
| | MLflow | 실험 추적 (SQLite) |
| | SHAP | 모델 설명 가능성 |
| **배포** | Vercel | 프론트엔드 (SPA) |
| | Railway | 백엔드 (FastAPI) |
