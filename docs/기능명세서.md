### F-08 · 생애 현금흐름 시뮬레이션 (`/simulation`)

| 항목 | 내용 |
|------|------|
| 입력 | `profile` (재무 프로필 전체) |
| 처리 | 35년간 월별 현금흐름 예측 + Claude AI 인사이트 생성 |
| 출력 | 현금흐름 분포형 차트, 핵심 요약, AI 인사이트, 경제지표 |
| API | `POST /api/simulation` |

**현금흐름 계산 로직**

```
[퇴직 전]
  소득 = 월소득 × (1 + 임금상승률)^연차
  지출 = 월지출 × (1 + 물가상승률)^연차 + 이벤트비용

[퇴직 후]
  소득 = 월연금 × (1 + 물가상승률×0.5)^퇴직후연차
  지출 = 목표생활비 × (1 + 물가상승률)^퇴직후연차 + 이벤트비용

[건강 이슈 보정]
  65세 이상: 추가 의료비 = 30만원 × (1 + 의료비상승률)^(나이-65)

[신뢰구간]
  σ = |현금흐름| × 0.08 × √연차  (누적 불확실성)
  상단 = 예측값 + σ
  하단 = 예측값 - σ
```

**적용 경제 지표 (최근 5년 평균, 한국 통계청)**

| 지표 | 값 |
|------|----|
| 소비자 물가 상승률 | 연 3.2% |
| 임금 상승률 | 연 3.5% |
| 의료비 상승률 | 연 4.5% |
| 불확실성 (±1σ) | 연 8% |

**AI 인사이트**: Claude API 호출로 핵심 리스크 1가지 + 시급한 행동 1가지 생성 (3~4문장, 평문)

### F-09 · 맞춤 추천 (`/recommend`)

| 항목 | 내용 |
|------|------|
| 입력 | `profile`, `surveyScores`, `diagnosis` |
| 처리 | 액션카드 생성(규칙 기반) + RAG 상품 검색 + Claude 선별 |
| 출력 | 우선순위 액션카드 최대 5개 + 맞춤 상품 3개 + 면책조항 |
| API | `POST /api/recommend` |

**액션카드 생성 규칙**

| 조건 | 생성 액션카드 |
|------|-------------|
| 재무 취약 + 국민연금 미납 | 국민연금 납부이력 점검 |
| 재무 취약 + 월 부족분 존재 | 노후 준비 적금 개설 |
| 개인연금 미가입 | 개인연금 3층 구조 완성 |
| 건강 취약 or 건강이슈 | 노후 의료비 리스크 대비 |
| 여가활동 취약 | 노후 여가 목적 자금 마련 |
| 대인관계 취약 | 가족·사회 연계 자산 준비 |
| 부동산 >1억 + 금융자산 <3천만 | 부동산 활용 유동성 확보 |

**RAG 상품 추천 프로세스**

```
1. 사용자 상황 → 자연어 쿼리 변환
   ex) "만 55세 직장인 / 노후 준비 취약 영역: 재무, 건강 / 리스크 성향: 중립형"

2. ChromaDB 벡터 검색
   - 실제 상품 top-4 + 가상 상품 top-3

3. Claude API 최종 선별
   - 후보 7개 중 최적 3개 선택
   - 각 상품별 추천 이유 2줄 생성
```

**상품 분류**

| 구분 | 내용 | UI 표시 |
|------|------|---------|
| 실제 상품 | 전북은행·광주은행 출시 상품 | 영역색 테두리 |
| 가상 상품 | JB금융 미출시 기획 상품 | 점선 테두리 + AlertTriangle 뱃지 |

### F-10 · AI 상담 (`/chat`)

| 항목 | 내용 |
|------|------|
| 입력 | 사용자 메시지 + 진단 결과 컨텍스트 + 대화 이력 |
| 처리 | Claude API 호출 (free 모드 / proactive 모드) |
| 출력 | AI 응답 텍스트, 추천 질문, 피드백 버튼 |
| API | `POST /api/chat` / `POST /api/chat/stream` |

**2가지 상담 모드**

| 모드 | 설명 |
|------|------|
| free | 자유 질문 응답 — 사용자가 원하는 노후 관련 질문 |
| proactive | 5단계 가이드 상담 — AI가 순서대로 핵심 질문 유도 후 종합 분석 |

**proactive 5단계 흐름**

1. 노후 준비 관련 주요 걱정사항
2. 현재 노후 자금 준비 현황
3. 목표 월 생활비
4. 보험 가입 현황
5. 예상 지출 이벤트 → 종합 분석 리포트 생성

### F-12 · MLOps 파이프라인

| 항목 | 내용 |
|------|------|
| 데이터 | KReIS 10차 (국민연금공단 노후준비실태조사, 8,736명) |
| 타겟 | 4대 영역 점수 (재무·건강·여가활동·대인관계, 각 0~100점) |
| 모델 | XGBoost / RandomForest 5-Fold CV 비교 선택 |
| 파이프라인 | DVC 3단계 (featurize → train → explain) |
| 실험 관리 | MLflow (SQLite 백엔드) |
| 설명 가능성 | SHAP TreeExplainer 피처 중요도 시각화 |

**모델 성능 (5-Fold CV)**

| 영역 | 최적 모델 | MAE | R² |
|------|----------|-----|-----|
| 재무 | RandomForest | 2.60 | 0.764 |
| 건강 | XGBoost | 8.42 | 0.530 |
| 여가활동 | XGBoost | 7.56 | 0.212 |
| 대인관계 | XGBoost | 10.78 | 0.317 |

## 5. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답 시간 | 시뮬레이션 < 5초, 추천 < 60초 (Claude API 포함) |
| 타임아웃 | Axios 60초 / Claude timeout 50초 |
| 보안 | CORS 도메인 지정, API 키 환경변수 관리 |
| 접근성 | 터치 영역 최소 52px, 폰트 최소 16px (고령 친화) |
| 면책 | 금융소비자보호법 준수 — 정보 제공 목적 명시 |
| 가상 상품 | UI에 반드시 AlertTriangle 뱃지 표시 |
