## 2. 화면 전환 다이어그램

```
/  ──────────────────────────────→ /consent
/consent ──[전체동의]──────────→ /area-select
/area-select ──[선택완료]───────→ /onboarding
/onboarding ──[연동완료]────────→ /survey
/survey ──[설문완료]────────────→ /pre-diagnosis
/pre-diagnosis ──[완료]─────────→ /dashboard
/dashboard ──[다음]─────────────→ /diagnosis
/diagnosis ──[시뮬레이션 보기]──→ /simulation
/simulation ──[맞춤 추천 받기]──→ /recommend
/recommend ──[AI에게 물어보기]──→ /chat

각 페이지 헤더의 ‹ 버튼으로 이전 화면으로 이동 가능
```

## 3. 데이터 수집 흐름

```
사용자 입력 수집 단계별 데이터:

[Onboarding] ──→ profile 초기값:
  · age, job_type, monthly_income, monthly_expense
  · financial_assets, real_estate_assets, liabilities
  · national_pension_expected, retirement_pension, personal_pension
  · spending_categories, accounts, insurance (JB연동 시)

[Survey] ──→ surveyScores:
  · { basic: 80, finance: 72, health: 60, leisure: 55, relation: 68 }

[PreDiagnosis] ──→ profile 보완:
  · retirement_target_age, monthly_target_living_cost
  · health_issue, life_events, risk_type

[AppContext에서 관리]
  ├── profile         → POST /api/simulation, /api/recommend, /api/chat
  ├── surveyScores    → POST /api/recommend
  ├── selectedAreas   → Diagnosis 차트 유형 결정
  ├── simulation      → Simulation 화면 렌더링
  └── recommend       → Recommend 화면 렌더링
```

## 4. 페르소나별 시나리오

### 시나리오 A — 김철수 (55세, 직장인)

```
특징: 재무 취약 + 생애이벤트 多 (자녀대학 + 부모부양)

예상 결과:
├── 진단: 재무 58점 (취약) / 건강 72점 (보통)
├── 시뮬레이션: 62세 은퇴 직후 적자 전환
│   · AI 인사이트: "개인연금 없음 + 이벤트 부담 → 즉각 IRP 가입 권장"
├── 추천:
│   · 액션카드 1: 개인연금 3층 구조 완성
│   · 액션카드 2: 국민연금 납부이력 점검
│   · 상품: JB 퇴직연금 IRP, JB 노후준비 적금
└── 상담: "은퇴 후 9년간 자금 공백 어떻게 메울까요?"
```

### 시나리오 B — 이영희 (53세, 자영업자)

```
특징: 부동산 집중 + 금융자산 부족 + 유동성 낮음

예상 결과:
├── 진단: 재무 55점 (취약) - 금융자산/연금 부족
├── 시뮬레이션: 자영업 소득 불규칙, 은퇴 시 급격 하락
│   · AI 인사이트: "부동산 편중 리스크 → 금융자산 전환 필요"
├── 추천:
│   · 액션카드 1: 부동산 활용 유동성 확보
│   · 액션카드 2: 노후 준비 적금 개설
│   · 상품: 주택연금, JB 적립식 예금
└── 상담: "자영업자 국민연금 추납 방법이 궁금해요"
```

### 시나리오 C — 박민준 (58세, 공무원)

```
특징: 연금 탄탄 + 건강 이슈 + 여가 미흡

예상 결과:
├── 진단: 재무 78점 (보통) / 건강 45점 (미흡) / 여가 52점 (미흡)
├── 시뮬레이션: 전 기간 흑자 (연금 충분)
│   · AI 인사이트: "의료비 리스크 대비 필요"
├── 추천:
│   · 액션카드 1: 노후 의료비 리스크 대비
│   · 액션카드 2: 노후 여가 목적 자금 마련
│   · 상품: 실손보험, JB 실버케어 건강적금(가상), JB 노후여가 적금(가상)
└── 상담: "건강 이슈가 있을 때 보험 추가 가입이 가능한가요?"
```

## 5. 오류 상황 처리 흐름

```
API 호출 실패 시:

[Simulation 페이지]
  API 오류 → AlertTriangle 아이콘 + 에러 메시지 + "다시 시도" 버튼
  → 버튼 클릭 시 simulation 상태 초기화 → 재호출

[Recommend 페이지]
  API 오류 → AlertTriangle 아이콘 + 에러 메시지 + "다시 시도" 버튼
  → 버튼 클릭 시 recommend 상태 초기화 → 재호출

[Chat 페이지]
  API 오류 → "응답 생성에 실패했습니다." 메시지 표시
  → 사용자가 다시 전송 가능

타임아웃 기준: Axios 60초 / Claude API 내부 50초
```
